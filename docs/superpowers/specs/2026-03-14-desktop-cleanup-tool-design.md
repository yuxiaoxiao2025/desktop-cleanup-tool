# 桌面整理小工具 — 设计文档

**日期**：2026-03-14  
**状态**：已确认

---

## 一、概述

常驻系统托盘的桌面整理小工具：监控桌面根目录，对新出现的文件/文件夹在**延迟时间**（默认 24 小时）后按 YAML 规则自动分类移动，并记录历史；支持从当前桌面“学习”目标结构与快捷方式白名单，规则可在设置界面中增删改。

---

## 二、功能摘要

- 系统托盘（右下角）图标，仅右键操作，无左键；悬停显示程序状态（是否在运行、是否暂停、待整理数量等）。
- 监控桌面根目录；新文件/新文件夹加入持久化待整理列表，满 **延迟时间**（小时）后按规则移动；移动后系统通知「XXX 已移至 XXX 文件夹」。
- 移动时若文件正在使用：不删除待整理项，增加重试计数，后续周期或「重试失败项」再次尝试；达到阈值时通知用户。
- 右键菜单：历史记录（近 10 条，格式「原名称 → 目标文件夹名」）、更多（打开全部历史页）、重试失败项、暂停/恢复监控、设置、从桌面学习、退出。
- 历史：持久化存储；「更多」在浏览器中打开，每条可点击跳转到目标路径（打开文件夹并选中项）。
- 设置界面：浏览器打开，编辑规则（增删改）、延迟时间、排除文件夹、快捷方式白名单等；延迟时间标注为「新文件/文件夹在桌面停留多久后再自动整理（小时）」。
- 规则：YAML 配置；内置默认规则 + 从当前桌面学习目标结构与快捷方式白名单；目标路径为相对桌面的路径。

## 三、规则与 YAML 结构

### 3.1 配置文件路径

- 配置：`%APPDATA%\DesktopCleanup\config.yaml`
- 首次运行无此文件时：用内置默认规则生成，并可选执行一次「从当前桌面学习」更新目标路径与白名单。

### 3.2 顶层结构

```yaml
desktop_path: "C:\\Users\\Administrator\\Desktop"
delay_hours: 24   # 新文件/文件夹在桌面停留多久后再自动整理（小时），建议范围 1–168

shortcut_whitelist:
  - "Cursor.lnk"
  - "Kimi.lnk"
  - "纳米AI.lnk"
  # ...

exclude_folders:
  - "00快捷方式"
  - "资料"

monitor_paused: false   # 与托盘「暂停/恢复监控」同步

rules:
  - name: "售后与统计-2025上半年统计"
    keywords: ["统计", "技术中控", "售后工作量"]
    extensions: [".docx", ".pdf", ".xlsx", ".xls"]
    target: "售后与统计\\2025上半年统计"
  # ...

shortcut_target: "00快捷方式"
default_target: "临时与杂项"
```

- `target` 为相对桌面的路径；匹配时按 `rules` 顺序，命中第一条即移动；都不命中则移到 `default_target`。
- 快捷方式：未在 `shortcut_whitelist` 的 .lnk 移入 `shortcut_target`。
- 「从桌面学习」：扫描桌面得到主题文件夹及子文件夹 → 更新可用的目标路径；桌面根目录的 .lnk → 覆盖 `shortcut_whitelist`。不自动生成规则条目的关键词/扩展名。

### 3.3 匹配逻辑

- 对每个待整理项（文件或文件夹名）：先判断是否为 .lnk → 是则用白名单与 `shortcut_target`；否则按 `rules` 顺序，文件名或文件夹名包含任一 `keywords` 或扩展名在 `extensions` 中即命中，移动至该条 `target`；都不命中则移到 `default_target`。

## 四、监控与 24 小时逻辑

### 4.1 监控对象

- 仅监控桌面根目录（`desktop_path`），不监控子文件夹。
- 「新」的定义：在桌面根目录新出现的文件或文件夹，且不在排除列表、不在待整理列表（或尚未被移走）。

### 4.2 创建时间与延迟

- 优先使用文件/文件夹的**创建时间**（CreationTime）；若无则用**最后修改时间**（LastWriteTime）。
- 「满延迟时间」：从创建时间（或 fallback）起算，超过 `delay_hours` 小时后，在下一轮扫描/执行时执行移动。

### 4.3 扫描与执行

- 采用「变更通知 + 定时复核」或纯定时扫描（如每 10 分钟）：对桌面根目录下每一项检查是否在待整理列表、创建时间是否已满 `delay_hours`。
- 到期则：解析目标路径（规则或白名单）、按需创建目标目录、执行移动、写入历史、从待整理列表移除、发通知「XXX 已移至 XXX 文件夹」。

### 4.4 排除

- 不监控、不移动：`exclude_folders` 中的文件夹、`desktop.ini` 等约定不移动项；正在被移动的项同一时刻只被一个进程/线程处理。

### 4.5 持久化待整理列表

- 路径：`%APPDATA%\DesktopCleanup\pending.json`
- 每项：`path`（绝对路径）、`name`、`created_at`、`added_at`、`retry_count`（可选）。
- 启动时加载并校验（路径不存在则移除）；扫描到新项则追加并写回；移动成功则删除该项并写回；移动失败则按 4.6 处理并写回。

### 4.6 文件正在使用时的处理

- 移动时若发生「文件正在被另一进程使用」类错误：不从待整理列表移除，`retry_count` 加 1，写回 `pending.json`。
- 后续周期或用户点击「重试失败项」时再次尝试移动。
- 当 `retry_count` 达到阈值（如 3）时，发一次系统通知：「XXX 因被占用未能自动移动，请关闭占用程序后等待下次自动重试，或在托盘菜单中点击“重试失败项”。」
- 不强制结束进程、不强制覆盖；仅重试 + 通知 + 手动重试入口。

## 五、托盘与右键菜单

### 5.1 托盘图标

- 位置：系统托盘（右下角）。
- 左键：无操作。
- 轻微区分（省内存）：不换图标，仅用 **tooltip** 区分状态。例如：
  - 监控中：`桌面整理 - 运行中` 或 `桌面整理 - 已监控`
  - 有待整理：`桌面整理 - 运行中（3 项待整理）`
  - 暂停：`桌面整理 - 已暂停`
- 悬停 tooltip 表达「是否在运行」及暂停、待整理数量等。

### 5.2 右键菜单

- **历史记录**：子菜单展示最近 10 条；每条格式 **「原名称 → 目标文件夹名」**；点击某条：打开目标所在文件夹并选中该文件/文件夹。
- **更多**：打开全部历史记录页面（浏览器）。
- **重试失败项**：对待整理列表中 `retry_count > 0` 的项再次尝试移动。
- **暂停监控 / 恢复监控**：切换 `monitor_paused`，写回 config；重启后保持。
- **设置**：打开设置页面。
- **从桌面学习**：更新目标结构与快捷方式白名单，写回 config。
- **退出**：退出程序。

## 六、历史与「更多」页

### 6.1 存储

- 路径：`%APPDATA%\DesktopCleanup\history.json`
- 单条：`moved_at`、`original_name`、`original_path`（可选）、`target_folder`（绝对路径）、`target_folder_display`（用于显示的目标文件夹名）。
- 仅追加；是否设上限（如最近 1000 条）可在后续或设置中配置。

### 6.2 近 10 条与「更多」页

- 从 `history.json` 按时间倒序取最新 10 条，在托盘「历史记录」中显示「原名称 → 目标文件夹名」；点击则打开目标文件夹并选中项。
- 「更多」：浏览器打开本地地址（如 `http://127.0.0.1:<port>/history`），列表展示全部历史；点击某条：在资源管理器中打开目标文件夹并选中该文件/文件夹（如 `explorer /select,"<目标路径>"`）。

## 七、设置界面

### 7.1 打开方式

- 托盘右键 →「设置」；浏览器打开 `http://127.0.0.1:<port>/settings`（与历史页同端口）。

### 7.2 页面内容

- **桌面路径**：显示/编辑 `desktop_path`；保存时校验存在且为目录。
- **延迟时间**：标签为 **「新文件/文件夹在桌面停留多久后再自动整理（小时）」**，数值范围建议 1–168，对应 `delay_hours`。
- **排除文件夹**：`exclude_folders` 列表；增删后保存写回 YAML。
- **快捷方式白名单**：`shortcut_whitelist` 列表；增删后保存。
- **快捷方式默认目标**：`shortcut_target`。
- **默认目标（未匹配时）**：`default_target`。
- **分类规则**：列表/表格展示 `rules`；每条可编辑、删除；支持新增（name、keywords、extensions、target）；target 可从当前 config 或「从桌面刷新」得到的候选路径中选择。
- **监控暂停**：勾选/开关对应 `monitor_paused`，与托盘「暂停/恢复监控」同步。

### 7.3 从桌面学习

- 设置页可提供「从当前桌面学习」按钮，行为与托盘一致；可选提示「已更新目标路径与白名单，请检查规则并保存」。

## 八、技术栈与实现要点

### 8.1 技术选型

- Python 3（建议 3.10+）
- 托盘：`pystray`
- 配置：`PyYAML`（config.yaml）、标准库 `json`（pending.json、history.json）
- Web：Flask 或 FastAPI，同一端口提供 `/settings`、`/history`
- 监控：`watchdog` 或定时轮询；实际执行仍以 `pending.json` + 创建时间 + `delay_hours` 为准
- 移动：`shutil.move`；捕获“文件正在使用”异常，按 4.6 处理
- 通知：Windows 系统通知（如 win10toast / plyer / winotify）

### 8.2 目录与单实例

- 数据目录：`%APPDATA%\DesktopCleanup\`
- 单实例：互斥量/锁文件或命名 Mutex，避免多进程重复监控与移动

### 8.3 可选

- PyInstaller 打包为单 exe
- 设置或安装时提供「开机自启」（注册表 Run）

## 九、规则来源说明（产品逻辑）

- **内置默认规则**：来自最初桌面整理方案（关键词/扩展名 → 目标路径），以及你调整后的主题与子文件夹（如 图纸与工程文件\\十里风和消防、图片与媒体\\大佛寺测试视频）。
- **从桌面学习**：用当前桌面的文件夹结构更新「目标路径」候选，用当前桌面根目录的 .lnk 更新「保留在桌面的快捷方式」白名单；不自动改规则条目的关键词/扩展名。
- **设置界面**：用户可对规则、延迟时间、排除项、白名单进行增删改，保存即写回 YAML。
