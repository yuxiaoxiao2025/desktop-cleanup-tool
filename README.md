# 桌面整理小工具

自动监控桌面，将超过设定「延迟时间」的文件/文件夹按规则整理到子目录；支持快捷方式白名单、从桌面学习目标结构、托盘菜单与 Web 设置页。可选**智能分类**：规则未命中时用反馈库或 Qwen 向量推荐目标，置信度不足时待用户确认，并支持从反馈批量生成规则建议。

## 安装与运行

1. **安装依赖**

   ```bash
   pip install -r requirements.txt
   ```

2. **环境变量（可选）**

   - 若启用**智能分类**（在设置页开启「智能分类」），需配置 `DASHSCOPE_API_KEY` 供 Qwen 向量接口使用；未设置时仅使用规则与反馈库，不调用远程 API。
   - 参考项目根目录下的 `.env.example`，复制为 `.env` 并填入密钥，或直接在系统中设置该环境变量。**若使用 `.env` 文件**，程序不会自动读取，需在启动前由系统/终端加载该文件，或使用 python-dotenv 等工具加载。获取 API Key：<https://help.aliyun.com/zh/model-studio/get-api-key>。

3. **启动程序**

   ```bash
   python main.py
   ```

   启动后会在系统托盘显示图标，无控制台窗口。若重复启动，会提示「已在运行」并退出（单实例）。

## 数据目录

配置与运行时数据存放在：

- **路径**：`%APPDATA%\DesktopCleanup`
- **内容**：首次运行会在此生成 `config.yaml`，以及待整理队列、历史记录等；启用智能分类并在发生采纳或用户确认后会生成 `feedback.json`（用户采纳目标与规则提炼用）。

端口等可从 `config.yaml` 中查看或修改，默认 HTTP 端口为 **57600**（避免与 Windows 保留端口冲突）。

## 首次使用与「从桌面学习」

- 首次运行若尚无 `config.yaml`，程序会先写入默认配置，并自动执行一次「从桌面学习」。
- **从桌面学习**：扫描桌面第一层目录（排除配置中的「排除文件夹」），得到目标路径候选（如 `父文件夹\子文件夹`），并扫描桌面上的 `.lnk` 文件名更新「快捷方式白名单」。可在托盘菜单中随时点击「从桌面学习」重新扫描更新。

## 智能分类与学习（可选）

- **解析顺序**：规则 → 反馈库（历史采纳）→ 向量推荐（需开启智能分类且配置 `DASHSCOPE_API_KEY`）。命中规则或反馈时直接采用；向量推荐置信度高于阈值时自动移动，低于阈值时进入**待确认列表**，在设置页确认或修改目标后再移动。
- **反馈与规则建议**：每次移动或用户确认后的目标会写入反馈库；在设置页可「从反馈生成规则建议」，勾选后一次性合并到配置，无需逐条编辑规则。

## 设置页与历史页

- **设置页**：<http://127.0.0.1:57600/settings>  
  配置桌面路径、延迟时间、排除文件夹、白名单、整理规则；可**从反馈生成规则建议**并采纳。智能分类开关与置信度阈值需在 `config.yaml` 中配置；待确认项可通过 `/api/pending-confirm` 接口查看与确认（设置页 UI 可后续提供）。
- **历史记录**：<http://127.0.0.1:57600/history>  
  查看已整理记录；也可在托盘菜单「历史记录」子菜单或「更多」中打开。

端口以 `config.yaml` 中的 `port` 为准（默认 57600），若修改了端口，请将上述 URL 中的端口号改为实际值。若出现「以一种访问权限不允许的方式做了一个访问套接字的尝试」，多为当前端口落在 Windows 保留范围内，请将 `config.yaml` 中的 `port` 改为 57600 或其他未保留端口。

## 延迟时间

- 在设置页中配置的「延迟时间（小时）」表示：**文件/文件夹在桌面上停留超过该小时数后，才会被自动整理**。
- 取值范围：1–168 小时。例如设为 24，则新放到桌面的项会在约 24 小时后才被移动到对应目标文件夹。

## 重试失败项与占用提示

- 若某文件因被其他程序占用而无法移动，程序会记录并在后续周期自动重试；重试次数达到约 3 次后会弹出 **被占用提示**，建议关闭占用程序后：
  - 等待下次自动重试，或  
  - 在托盘菜单中点击 **「重试失败项」** 立即再试。

## 托盘菜单说明

| 菜单项       | 说明 |
|--------------|------|
| 历史记录     | 子菜单列出最近整理记录，点击可在资源管理器中定位到目标文件。 |
| 更多         | 在浏览器中打开历史记录页。 |
| 重试失败项   | 对因占用等原因未移动成功的项再次尝试移动。 |
| 暂停/恢复监控 | 暂停或恢复自动整理，配置会保存。 |
| 设置         | 在浏览器中打开设置页。 |
| 从桌面学习   | 根据当前桌面目录与快捷方式更新白名单与目标路径候选。 |
| 退出         | 退出程序。 |

托盘图标悬停会显示「运行中」或「已暂停」「待整理项数量」等状态。

## 打包为 exe（可选）

若希望生成单个 exe 便于分发，可使用 PyInstaller：

```bash
pip install pyinstaller
pyinstaller --onefile --windowed main.py
```

- `--onefile`：打成单个 exe。  
- `--windowed`（或 `--noconsole`）：不显示控制台黑窗，适合托盘程序。

**注意**：需保证 exe 能找到 **icon.png**、**templates/** 等数据文件。可任选其一：

1. 在 spec 中通过 `datas` 将 `icon.png`、`templates` 等打包进 exe；或  
2. 将 exe 与 `icon.png`、`templates` 目录放在同一目录下，程序从运行目录读取。

打包为可选步骤，不要求必须执行。
